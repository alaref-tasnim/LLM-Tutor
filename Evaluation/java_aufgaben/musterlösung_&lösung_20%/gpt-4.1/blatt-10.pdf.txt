import java.util.ArrayList;

public class HashTableChaining<K, V> implements HashTable<K,V> {

    private static final int SIZE = 97;

    private ArrayList<HashEntry<K,V>>[] array;

    @SuppressWarnings("unchecked")
    public HashTableChaining() {
        this.array = new ArrayList[SIZE];
    }

      @Override
    public void insert(K key, V value) {
        if (value == null) {
            throw new NullPointerException("value must not be null");
        }
        int h = hashValue(key);
        ArrayList<HashEntry<K,V>> l = this.array[h];
        if (l == null) {
            l = new ArrayList<>();
            HashEntry<K, V> entry = new HashEntry<>(key, value);
            l.add(entry);
            
        } else {
            for (HashEntry<K,V> e : l) {
                if (e.getKey() == key) {
                    e.setValue(value);
                    
                }
            }
            HashEntry<K, V> entry = new HashEntry<>(key, value);
            l.add(0, entry);
        }
    }


    private int hashValue(K k) {
        int hc = k.hashCode();           // Hashcode
        int h = Math.abs(hc) % SIZE;       // Hashwert, Index im Array
        return h;
    }

    @Override
    public V lookup(K key) {
        int h = hashValue(key);
        ArrayList<HashEntry<K,V>> l = this.array[h];
        
         
        for (HashEntry<K,V> entry : l) {
                if (entry.getKey().equals(key)) {
                    return entry.getValue();
            }
        }
        return null;

### 1. Funktion: insert (Zeilen 13-36)

#### Vorhandene Fehler:
- **Zeile 19/20:** Initialisierte Liste `l` wird nicht im Array gespeichert. Es fehlt die Zuweisung `this.array[h] = l;`, wodurch die neu erstellte Liste niemals in das Array eingetragen wird.
- **Zeile 26:** Es wird mit `==` verglichen (`e.getKey() == key`). Für Objekte (inklusive generischer Typen wie K) muss stattdessen `.equals()` verwendet werden, da `==` nur prüft, ob es sich um dasselbe Objekt handelt, nicht ob sie inhaltlich gleich sind.
- **Zeile 31:** Nach Setzen des neuen Werts wird dennoch ein neuer Eintrag mit demselben Schlüssel eingefügt, was zu doppelten Schlüsseln führen kann.
- **Algorithmus:** Es wird nicht geprüft, ob der Schlüssel bereits vorhanden ist, BEVOR ein neuer Eintrag hinzugefügt wird. Der Eintrag wird immer hinzugefügt, selbst wenn der Schlüssel schon existiert. Laut Aufgabenstellung soll ein vorhandener Wert nur geändert, nicht ein neuer Eintrag erzeugt werden.
- **Geforderter Algorithmus / Abweichung:** Es ist Hashing mit Chaining gefordert. Die grundlegende Idee (String-zu-Hash, dann Zugriff auf die Liste im Array) ist zwar verwendet, aber der Einfüge- und Ersetz-Mechanismus ist fehlerhaft: 
  - **Gefordert war:** Ketten (Listen) nur um neuen Schlüssel zu ergänzen, wenn er nicht existiert; ansonsten Aktualisierung.
  - **Implementiert wurde:** Ketten werden nicht richtig ins Array gespeichert und bei Überschreibung kommt es zu doppelten Schlüsseln durch immer neues Hinzufügen.
  
#### Fazit:
Die Methode ist fehlerhaft (siehe oben) und entspricht nicht vollständig dem geforderten Algorithmus.

### 2. Funktion: lookup (Zeilen 43-52)

#### Vorhandene Fehler:
- **Zeile 45:** Es wird nicht geprüft, ob `l` null ist, bevor in der for-each-Schleife darauf zugegriffen wird. Falls noch nie ein Element in diese Liste eingefügt wurde, ist sie `null`, was zu einer NullPointerException führt.
- **Algorithmische Korrektheit:** Bis auf die fehlende Null-Prüfung sucht die Methode korrekt mit `.equals()` nach einem Schlüssel in der Liste an der gehashten Position.
- **Geforderter Algorithmus / Abweichung:** Hashing mit Chaining ist hier grundsätzlich (vom Grundprinzip) korrekt angewandt.

#### Fazit:

### 3. Funktion: remove (Zeilen 55-64)

#### Vorhandene Fehler:
- **Zeile 57:** Falscher Hashwert! Es wird immer `this.array[3]` verwendet, anstatt `this.array[h]`. Das bedeutet, dass für JEDEN Schlüssel nur versucht wird, in der Liste am Index 3 zu löschen, unabhängig vom eigentlichen Schlüssel.
- **Zeile 59/61:** Es wird während der Iteration über die Liste ein Element daraus entfernt (`l.remove(e);`). Dies kann zu ConcurrentModificationException führen, da es keinen Iterator gibt und in einer for-each-Schleife entfernt wird.
- **Algorithmische Korrektheit:** Selbst wenn die Zeile mit dem festen Index 3 korrekt wäre, würden durch das direkte Entfernen Fehler auftreten.
- **Geforderter Algorithmus / Abweichung:** Hashing mit Chaining ist grundsätzlich beabsichtigt, aber durch fehlerhafte Array-Indexierung und Listenzugriffe (siehe oben) nicht korrekt umgesetzt.

#### Fazit:
Die Methode ist fehlerhaft und entspricht auch nicht dem geforderten Algorithmus (korrektes Hashen, Suche und Entfernen in der richtigen Kette).

### 4. Funktion: size (Zeilen 67-72)

#### Vorhandene Fehler:
- **Fehlerhafter Algorithmus:** Die Methode gibt bereits bei der ersten nicht-leeren Liste die Größe dieser Einzelkette zurück (Zeile 69), anstatt die Größe aller Listen (also insgesamt aller HashTable-Einträge) zu liefern.

- **Geforderter Algorithmus / Abweichung:** Es sollte eine Schleife über alle Ketten laufen, um die Summe aller Einträge zu ermitteln. Hier wird nur die erste belegte Kette betrachtet.

#### Fazit:
Die Methode ist funktional und logisch inkorrekt, berechnet nicht die Gesamtgröße der Hashtabelle und entspricht nicht dem geforderten Algorithmus.

Unterschiede:

**insert**
- Musterlösung: Weist beim Erstellen einer neuen Liste explizit this.array[h] = l; zu. Wenn der Key bereits existiert, wird nur dessen Wert geändert und sofort return benutzt.
- Studentenlösung: Weist der Arrayposition keine Liste zu (vergisst this.array[h] = l;), überschreibt nie return, kann so mehrfach denselben Key einfügen, prüft Key-Vergleich mit == statt .equals().

**lookup**
- Musterlösung: Prüft zunächst auf null, durchläuft dann mit .equals() die Liste.
- Studentenlösung: Prüft nicht, ob l==null (würde zu NullPointerException führen), benutzt korrekt .equals() beim Vergleich.

**remove**
- Musterlösung: Nutzt korrekten Hashindex (this.array[h]); iteriert mit .equals(), entfernt nur das erste passende Element und kehrt dann zurück.
- Studentenlösung: Nutzt statisch this.array[3] (statt this.array[h]), durchsucht also immer nur Index 3, vergleicht korrekt mit .equals(), kehrt aber nicht nach dem Entfernen zurück.

**size**
- Musterlösung: Zählt alle Listenelemente im Array auf und gibt die Gesamtzahl zurück.
- Studentenlösung: Gibt nur die Größe der ersten nicht-leeren Liste zurück (return in der Schleife), bricht also zu früh ab.

**Feedback:**

Die Studentenlösung zeigt grundlegende Schwächen bei Listeninitialisierung, Arrayzugriff und Terminierung von Rückgaben; außerdem werden Schlüssel teils nicht korrekt verglichen (== statt .equals()). Das führt zu fehlerhaftem Verhalten in fast allen Methoden und ist damit weniger sinnvoll und schlicht fehlerhaft im Alltagsgebrauch.
